Virtual TCP/IP Network Stack

Sep 2024 – Nov 2024

This project was completed as part of Brown's Computer Networks course in collaboration with Keven Li. Out of respect for the course, we have not posted the source code. Please contact me if you would like more information.

A significant portion of Computer Networks at Brown is a large, extended project in which students start from scratch and implement IP and then on top of that TCP, ending up with a router program which can route packets to the correct destination over a self-healing network and a host program which can send and receive data reliably.

The project builds off of a UDP "Layer 2", in which sending data out from a "NIC" results in the segment getting transferred over loopback UDP to the other node, which is just another instance of the program running on the same machine.

Internet Protocol (IP)

We implemented IP to support packet forwarding via longest-prefix matching, updating relevant IPv4 header fields correctly (e.g. TTL, checksum), and automatic route discovery / healing using RIP with split horizon and poison reverse. This allows networks using our IP stack to remain usable even when an intermediate node goes down (as long as there are redundant connections).

  • ● Longest-Prefix Matching: Implemented packet forwarding using longest-prefix matching to determine optimal routing paths
  • ● IPv4 Header Management: Correctly updated and validated IPv4 header fields including TTL (Time To Live) and checksums
  • ● RIP with Split Horizon and Poison Reverse: Built automatic route discovery and network healing capabilities, enabling self-healing networks that adapt to topology changes

Transmission Control Protocol (TCP)

We implemented TCP by directly referencing RFC 9293. We built support for TCP handshake, teardown, retransmissions over lossy connections, handling out-of-order segments, and zero-window probing.

  • ● TCP Handshake & Teardown: Implemented the complete TCP connection lifecycle including three-way handshake for connection establishment and proper connection termination
  • ● Retransmissions: Built reliable data transfer with automatic retransmission mechanisms to handle packet loss over lossy connections
  • ● Out-of-Order Segment Handling: Implemented proper sequencing and reassembly of TCP segments that arrive out of order
  • ● Zero-Window Probing: Developed flow control mechanisms including zero-window probing to handle receiver buffer constraints

Development & Debugging

Throughout implementing both IP and TCP, we used Wireshark as a primary debugging tool, to ensure checksums were correct, headers were well-formatted, retransmissions were behaving correctly, and ACK numbers were updating as expected especially in the case of dropped packets.
