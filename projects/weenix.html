Weenix Operating System

Jan 2025 – May 2025 Brown CS 1690
Learn More About Weenix

Developed major components of a Unix-like operating system kernel ("Weenix"), writing 9,000+ lines of C implementing threads, processes, device drivers, the virtual file system interface, and the underlying file system.

Core Components

Threads & Processes

Implemented process creation, scheduling, and termination with context switching and proper resource cleanup

Synchronization Primitives

Developed mutexes, condition variables, and semaphores to ensure safe context switching and reliable concurrency within the kernel

Device Drivers

Implemented drivers for keyboard, console, and timer devices, handling hardware interrupts and providing clean interfaces for system interaction

File Systems

Built a hierarchical file system with directory support, file operations (create, read, write, delete), and proper inode management

Virtual Memory

Implemented user address space management, paging, virtual address translation, and memory management with proper handling of page faults

System Calls

Created a comprehensive system call interface allowing user programs to interact with kernel functionality safely

Technologies

C Operating Systems Systems Programming Kernel Development

Reflection

Working on Weenix was one of the most challenging and rewarding experiences of my computer science education. Building an operating system from the ground up gave me a deep understanding of how computers actually work beneath the abstractions we typically use.

The project taught me the importance of careful memory management, understanding system architecture, and debugging complex concurrent systems. It was incredibly satisfying to see everything come together and watch user programs run successfully on an operating system that I helped build.
