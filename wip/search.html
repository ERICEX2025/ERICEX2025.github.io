Search

Apr 2022 – May 2022 Brown CSCI0200

As part of Brown University's Data Structures and Algorithms course (CSCI0200), I developed a search engine that processes a corpus of documents to efficiently handle user queries. The project involved building an indexer to preprocess documents and create efficient data structures, a querier to handle user queries using the preprocessed data, and implementing the PageRank algorithm to enhance search result relevance. The system processes XML-formatted documents and supports complex query operations with optimized time and space complexity.

Indexer

Implemented an indexer that processes a corpus of XML documents to build efficient data structures for querying. The indexer parses XML files, extracts document content and metadata, tokenizes text, and creates an inverted index mapping terms to documents. I designed the data structures to optimize both time and space complexity, using hash maps and other appropriate collections to enable fast lookups during query processing. The indexer handles edge cases in XML parsing and text processing.

Querier

Built a querier that processes user queries using the preprocessed index. The querier handles various query types, including single-term queries, multi-term queries, and complex boolean operations. I implemented efficient query processing algorithms that leverage the inverted index to quickly find relevant documents. The querier combines term-frequency and inverse document frequency (TF-IDF) scoring with PageRank to rank search results by relevance. The system returns results in order of relevance, with proper handling of edge cases and query parsing.

PageRank Algorithm

Implemented the PageRank algorithm to compute document importance based on link structure. The algorithm models documents as nodes in a graph where links between documents represent edges. I implemented iterative PageRank computation using power iteration, handling dangling nodes and convergence criteria. The PageRank scores are combined with TF-IDF scores to create a composite relevance ranking that considers both content relevance and document importance. The implementation efficiently processes large document collections and handles edge cases in graph structure.

Reflection

The Search project was a comprehensive introduction to information retrieval and search engine architecture. Building the indexer taught me the importance of choosing appropriate data structures for specific problem constraints—hash maps for fast lookups, efficient text processing, and careful consideration of space-time trade-offs. Implementing the querier gave me hands-on experience with ranking algorithms and query processing, reinforcing the importance of algorithmic efficiency in real-world applications.

Implementing PageRank was particularly rewarding, as it demonstrated how graph algorithms can be applied to solve practical problems. The iterative nature of PageRank computation and the need to handle edge cases like dangling nodes taught me to think carefully about algorithm correctness and convergence. Combining multiple ranking signals (TF-IDF and PageRank) showed me how real search engines balance different relevance factors. This project deepened my understanding of data structures, algorithms, and their practical applications in information retrieval systems.
